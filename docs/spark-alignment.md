# Spark Alignment

PureFlow solves a personal problem: regular use of coding agents can crowd out the manual repetitions that build code-reading, debugging, API recall, and ownership. The practical feature is a working IDE session, not a token or a speculative protocol.

## Submission fit

- Practical impact: a developer can run a real Rep in an existing repository and leave with evidence they can explain.
- Live product: every primary control changes persisted local state; Knowledge Dock uses real sources; the contract has tests and will be deployed on Monad Testnet.
- Onchain component: a completed Rep may be attested through `RepRegistry` using a privacy-safe commitment.
- Distinct identity: the interface is a restrained training console, not a tutorial dashboard or AI-generated card grid.
- Runnable in three minutes: the repo will include install, launch, demo, contract, and verification commands plus a demo workspace.

## Required submission fields

- Name: PureFlow
- Category: Monad Testnet
- Contract address: filled after deployment and bytecode verification
- Project URL: public landing/demo surface plus downloadable VSIX/distribution instructions
- GitHub: https://github.com/yava-code/PureFlow
- Demo video: three-minute script and recording checklist will be included
- Social post: copy-ready launch post will be included

## Honest limits

PureFlow is voluntary focus infrastructure, not anti-cheat. The onchain attestation proves a wallet published a commitment; it does not prove how the developer worked. The product does not claim that one session objectively increases skill.
