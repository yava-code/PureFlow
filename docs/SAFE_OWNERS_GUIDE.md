# Гайд: 2 адреса для Safe (актуально)

Нужны **не рандомные** адреса, а **2 публичных EVM-кошелька, которыми ты управляешь** (MetaMask / Rabby / OKX и т.п.).

Они станут **owners** 2-of-3 Safe вместе с agent-кошельком. Потом ты **подпишешь** деплой `RepRegistry` в Safe UI.

**Никогда не присылай** seed phrase, private key, JSON keystore.

---

## Зачем именно 2

| Роль | Кто | Адрес |
|------|-----|--------|
| Owner 1 | **Ты** (кошелёк A) | твой `0x…` |
| Owner 2 | **Ты** (кошелёк B) | второй твой `0x…` |
| Owner 3 | Agent (уже есть) | `0xe0D9466626be495C8ECC339E6866f72E9dad06C9` |

**Порог: 2 из 3** — для исполнения tx нужны любые 2 подписи.  
Agent + один твой, или оба твоих — ок.  
Без твоих подписей контракт **не** задеплоим (Monskills).

---

## Какой формат адреса правильный

**Нужно (Monad = EVM):**
```text
0x + ровно 40 символов 0-9 a-f / A-F
```
Пример формата (это пример, не копируй чужой вслепую):
```text
0xbcE7b3eB6Cf8b23957547FbCCA21333bB85CB1A7
```

**Не подходит:**
| Что | Почему |
|-----|--------|
| `4Mq57n7…` (Solana) | Другая сеть, Safe на Monad не примет |
| ENS-имена без resolve | Нужен raw `0x…` |
| Exchange deposit address | Не сможешь подписать Safe |
| Случайный `0x` из интернета | Не сможешь approve |
| Private key / seed | **Никогда** |

## Сеть

- **Monad Testnet**
- **Chain ID:** `10143`
- **RPC:** `https://testnet-rpc.monad.xyz`
- **Explorer:** https://testnet.monadscan.com  
- **Faucet (если понадобится MON):** https://faucet.monad.xyz  

Для **роли owner** (только подпись) баланс часто **не обязателен**.  
Gas при создании Safe / propose обычно платит **agent** (у него уже ~1 Testnet MON).  
Если Safe UI попросит gas с твоего — возьми testnet MON с faucet на **тот** адрес.

## Как получить 2 адреса (MetaMask) — самый простой путь

### 1) Установи MetaMask

https://metamask.io → браузерное расширение → создай / импортируй **свой** кошелёк.

### 2) Добавь Monad Testnet

MetaMask → сеть → **Add network** / Add a custom network:

| Поле | Значение |
|------|----------|
| Network name | `Monad Testnet` |
| RPC URL | `https://testnet-rpc.monad.xyz` |
| Chain ID | `10143` |
| Currency symbol | `MON` |
| Block explorer | `https://testnet.monadscan.com` |

Сохрани и переключись на **Monad Testnet**.

### 3) Адрес №1

1. В шапке MetaMask кликни имя аккаунта.  
2. **Account 1** (или текущий).  
3. Иконка копирования адреса → получишь `0x…`.  
4. Это **OWNER_1**.

### 4) Адрес №2 (второй аккаунт в том же MetaMask)

1. Клик по кружку аккаунта → **Add account** / Create account.  
2. Имя, например `PureFlow Safe 2`.  
3. Открой новый аккаунт → **Copy address**.  
4. Это **OWNER_2**.

Оба адреса — **твои**, оба EVM, оба подходят. Два отдельных seed **не** нужны.

### Альтернатива

- Второй аккаунт в **Rabby** / другой MetaMask (другой профиль браузера).  
- Главное: ты можешь открыть **https://app.safe.global** под каждым `0x…` и подписать.

---

## Что прислать агенту (одна строка)

```text
OWNER_1=0x........................................
OWNER_2=0x........................................
```

Пример с уже валидным первым (если это **твой** MetaMask):

```text
OWNER_1=0xbcE7b3eB6Cf8b23957547FbCCA21333bB85CB1A7
OWNER_2=0xВСТАВЬ_ВТОРОЙ_ИЗ_METAMASK
```

Solana `4Mq57…` **не** используем.

---

## Что будет после того, как пришлёшь оба `0x`

1. Agent создаст Safe 2-of-3 (tx с agent-кошелька).  
2. Сохранит `~/.monskills/multisig.json`.  
3. Запустит `propose.sh` на деплой `RepRegistry` → QR / ссылка Safe.  
4. **Ты** открываешь Safe UI, подключаешь MetaMask (**OWNER_1** или **OWNER_2**), **Confirm / Execute**.  
5. Присылаешь **execution transaction hash**.  
6. Проверяем bytecode → получаем **contract address** → вставляешь в форму Spark.

---

## Мини-чеклист перед отправкой

- [ ] Оба адреса начинаются с `0x` и длина 42 символа  
- [ ] Это **твои** кошельки (можешь подписать)  
- [ ] Не Solana / не CEX deposit  
- [ ] Не private key  
- [ ] Понимаешь: без твоей подписи в Safe деплой не завершится  

---

## Частые ошибки

1. **«Давай 2 рандомных»** — Safe создастся, approve сделать некому → contract address не появится.  
2. **Один EVM + один Solana** — второй отбросим.  
3. **Два одинаковых 0x** — плохо для 2-of-3; нужен второй **разный** аккаунт.  
4. **Прислал seed «на всякий»** — не надо; только public `0x`.
