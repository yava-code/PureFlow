# PureFlow End Goal

## North star

A senior developer can install or launch PureFlow, open a real repository, and use it as their primary IDE without feeling that they entered a tutorial. They write and debug in the native editor, call a bounded mentor only when wanted, practice deliberately when useful, and inspect or prove Monad work with live, verifiable data.

## The finished experience

1. Launch the branded PureFlow portable VSCodium distribution.
2. Open an existing folder, clone a repository, or create a normal project or Monad starter.
3. Work through the native editor, Explorer, search, Git, terminal, tests, debugger, and extension ecosystem.
4. Select code and invoke Explain, Explain why, Quiz me, Find docs, or Review reasoning from the editor context menu, command palette, keyboard, or compact sidebar.
5. Keep every AI request explicit and bounded to the visible selection or current function. A local guide remains useful when no coach endpoint is configured and is labeled honestly.
6. Optionally start a Focus Rep to work manually, record hypotheses and evidence, and complete a short defense.
7. Inspect Monad Testnet health, addresses, transactions, and project readiness from the IDE using live RPC.
8. Prepare a privacy-safe Rep commitment locally, hand it to the web companion, approve a wallet transaction, wait for a real receipt and safe/finalized state, and verify the commitment publicly.
9. Reproduce the distribution, extension, web companion, contract tests, demo, and submission from the public repository.

## Definition of done

- No runtime path calls `createWebviewPanel` for the PureFlow workbench.
- Opening PureFlow reveals a WebviewView sidebar and leaves the active editor untouched.
- Open/create folder, terminal, tests, source control, and editor context actions work through native VSCodium commands.
- Sidebar routes render correctly at 240, 320, and 420 pixels and at 200% zoom.
- Mentor uses a bounded selection/current function, never background repository contents, and is blocked during an active Focus Rep.
- Focus Rep remains fully usable but is not the startup screen or a prerequisite.
- Monad inspector reports chain ID 10143 and live address/transaction data with timeouts and explorer links.
- Project Doctor detects Hardhat, Foundry, Solidity, and frontend Monad signals without changing the repository.
- Any onchain `Verified` label is backed by a successful receipt and a follow-up contract read.
- Unit, build, packaging, web, contract, and portable smoke checks pass.
- Public GitHub Pages, repository, release/download instructions, Spark submission text, and demo script describe the same product.

## Explicit non-goals

- Maintaining a deep source fork of VSCodium before upstream-compatible branding proves insufficient.
- Replacing mature native editor, terminal, debugger, Git, or test surfaces with React copies.
- Becoming a coding course, challenge catalog, developer ranking system, wallet dashboard, token, NFT, or staking product.
- Autonomous patch generation or silent application of AI changes.
- Uploading entire repositories or storing sensitive practice content onchain.
- Claiming that one score or Rep proves developer skill.

## Post-hackathon direction

After the reproducible open-source release is stable, PureFlow may add cross-platform installers, signed releases, richer language-service context, team-owned proof policies, optional local-model providers, and an indexed public proof history. These remain secondary to being a calm, trustworthy daily IDE.
